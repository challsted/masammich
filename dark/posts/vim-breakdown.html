As you should know, vim is a Modal Commandline Text Editor. Which, as I understand it, means that it uses modes, i always think of it as a "mode cli text editor". So it uses "modes" to edit or add text. If you have zero expierence with vim, and you have it installed. Just type vim into your terminal. Beginner Commands

VIM Has "Modes"
So first you need to understand what "mode" your in and what those modes mean.

If you just opened vim, your in "Command" Mode.
In this mode all keystrokes are not interpreted as the letter you press. They all have meanings that we will learn soon.
You will know your in command mode when you do not see anything inside of brackets down in the bottom left.

Next you have "Insert" Mode.
The easiest way to get to this mode is by hitting "i", but there are several other ways to get to insert mode.
You will know your in insert mode when a small "[INSERT]" appears at the bottom left of your screen.
When you are in Insert mode, all of your keystrokes are not interpreted, instead they are taken as you type.
So if you hit the "i" key it types "i" instead of changing your mode.

Next you have "Replace" mode.
The only way I know to get into insert mode is by pressing "R" (Capitol R). You can also go into temporary replace mode by pressing "r" (lowercase r)
You will know you are in replace mode when you see a small "[REPLACE]" appear at the bottom left of your screen.
When you are in Replace mode, any key you type will replace what is underneth the cursor. So if your cursor is hovering over a P and you press the T key, that O will be replaced by a T.
"Temporary" replace mode (as I refer to it) will only replace one character (the one under the cursor) then return you to command mode.

Last mode you need to learn is "Visual" mode.
You can get to Visual mode a few differnt ways, but each way preforms a differnt type of "visual" task.
"Shift-V" puts you into "Visual Line" mode, this allows you to highlight single characters or full words.
"^V" (This means "Leader V" which by default is CTRL V) puts you into "Visual Block" mode, this allows you to edit whole lines at a time, useful to highlight full blocks of code or text.
You will know your in visual mode when you see "[VISUAL]" appear at the bottom left of your screen.
Visual Line and Visual Block are ment for High-Lighting text, I believe the only purpose of this is to "yank" (aka Copy) text

How to enter commands into VIM

Next you need to know when to enter commands, and how to leave differnt modes.
As I already said, by default your in "Command Mode", you have to go into other modes by pressing a a key, or a combo of keys.

The "Escape" key (top left most key on all standard keyboards usually marked as "Esc") is ment to get you out anything.
If your ever in a strange mode, or if you just want to go back to command mode from Insert mode. You can hit the trusty esc key to get out of almost any mode.

Every key on the keyboard has some kind of a command tied to it, and most of them have 2 or more commands tied to them, so a command could be used by hitting the key, hititng shift then the key, hitting alt then the key("ALT" or <ALT>), hitting ctrl then the key ("^" or "CTRL" or <CTRL>), or hitting "leader" then the key which is "\" by default (<leader>).

Now you need to know about ":" or "colon" commands.
Colon commands can only be entered from command mode. You enter them by typeing the colon key (":") then what ever the command is after the colon.

Commands we need to know first is how to quit VIM. The following commadns all "quit" vim. Key word is "quit" not "exit".
:q, :q!, ZZ
So, thats ":q" I usually hear people pronounce this as "Colon Q". As mentioned above, this is only entered form command mode.
Please note that ZZ is 2 capitoal Z's.

Now you need to know how to "write" your file, again key word is "write" not "save".
:w
But what if you want to Write the file and quit? Do you need to run :q then :w? NO! Vim allows for these 2 to be preformed at the same time, you can do this by typeing them together. like this:
:wq so you "write" then "quit". Do not try to ":qw" that would "quit" then it would never write the document because it quit...
What if you need to save the file to a differnt location, or in a differnt name?
:w ~/other/folder for writeing to a differnt location, or :w differntName.txt to write it as a differnt name (like "save as").
You can also combine this as :w ~/location/to/a/place/notTheOrgionalName.sh

PLUG-INS!

So, VIM comes with a lot of features, and thing that can be done by default.
But as you have seen above, some times default isnt good enough, we play with settings to make it better or just personalized.
Now, some people said that this still isnt enough! So they submitted various features to the owner of VIM and were rejected, so rather then give up. They (and others) started makeing "plug-ins" to add functionality on top of default VIM. This action caught on and is now very popular in the VIM community (very few people DONT use plugins in fact) and there are now Plugins to do almost anything you can imageine (and more frequently being created).

How do I install these?

There are several ways to do this. You can do it manually, or you can use a plugin manager (recomended) to do it all for you.

Manual although there is nothing wrong with doing it this way, if you plan on having more then a small number of plugins, it might be a good idea to use a plugin manager. But I will still list this way for research, and knowledge.

So first you need to create a directory, then a few directorys underneth that, I'm not sure if this is 100% needed, but the way I will list here, is the way i was taught, and i always see others doing it this same way (even with a manager).
So, first create a folder called ".vim" at your home directory.
mkdir ~/.vim
Now go into that directory and we need to create several more dirctorys, I'll explain what each one is for so you can not create them or just remove them if you dont want them.
mkdir -p ~/.vim/{autoload,colors,plugged,swaps,spell}
The above will create 5 sub-directorys underneth ~/.vim. Each of these direcorys has a specific purpose.
autoload - If you want a certain file to load as soon as vim loads (at the same time) this is where you want to put the .vim file. This is mainly used to put what ever plugin manager you use (you put the manager's .vim file here) so you may or may not want it. Im leaving it in here because Im going to assume you want to just learn how it works, not actually doing allt his your self.
colors - This is used for all of your color schemes, being 100% honest, I have 179 colorschemes located in this file right now... I dont even know how I got this many... I probably downloaded a zip of them when i was playing with vim and forgot ... (i have several files still here that dont see to have been touched in ~a year...). I will suggest you wget or git clone your favorate colorscheme into this folder. Once vim sees you have a .vim, it seems to start referenceing it from that point on. So forexample, i LOVE the molokai color scheme, so i cloned it into my ~/.vim/colors dir so i can reference it with my .vimrc.
plugged - Put all of your pluggin files here
swaps - You need to set in your .vimrc to use this for swap files. Its more convient then having random .~filename floating everywhere you forgot to save or quit properly and left a shadow file behind...
spell - If you chose to turn on spell check in vim, this is where it will save your spell file. Which is where it saves all the words you add or tell it to ignore in a little file.

Thats about it, you either wget or git clone the .vim file's into the appropriate place. Colorscheme into the "colors" dir, plugins into the "plugged" dir, files to load on launch into the "autoload" dir and the others you setup to have files put into automatically.

Manager This is highly recamended, it makes your life far easier due to it takes the process of updateing, and placeing files where they need to go out of the picture. Please see the individual links on how to install the sepearte plugin, the process changes over time, and I dont want to be behind a revision or 2 for how to install.

Vim-Plug
I personally use this manager. I use because it works in both vim and NeoVim.
It installs and updates in paralell.
It allows you to load specific branch or a tag of a plugin, so you can play with experimental versions of a plugin

Pathogen
One of the strengths of Pathogen, is that it creates seperate folders for every plugin, and inside of that folder, is everything the plugin needs to work.
The benefit of this is that 1 plugin wont effect the others you have.
Another benefit of this is that if you want to remove a plugin, you simply remove that directory that is named after the plugin.

Vundle
The Benefit of Vundle is that it is actually built on top of pathogen, so you get all the benefits of pathogen, but you also get a management interface within vim. This is the most popular vim plugin manager that I know of, most plugins you find on GitHub will have vundle instructions on how to use it.
