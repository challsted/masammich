VIM Commands

As you should know, vim is a Modal Command line Text Editor. Which, as I understand it, means that it uses modes, I always think of it as a "mode cli text editor". So it uses "modes" to edit or add text. If you have zero experience with vim, and you have it installed. Just type vim into your terminal. It will open a "vim tutor" which attempts to teach you vim, you also have the option of playing the game I mentioned before which can be found at VimAdventures.com thats pretty much a TLDR of the previous article. So let's get on with the VIM Commands shall we!

Beginner Commands

VIM Has "Modes"
So first you need to understand what "mode" your in and what those modes mean.

If you just opened vim, you're in "Command" Mode.
In this mode all keystrokes are not interpreted as the letter you press. They all have meanings that we will learn soon.
You will know you're in command mode when you do not see anything inside of brackets down in the bottom left.

Next you have "Insert" Mode.
The easiest way to get to this mode is by hitting "i", but there are several other ways to get to insert mode.
You will know you're in insert mode when a small "[INSERT]" appears at the bottom left of your screen.
When you are in Insert mode, all of your keystrokes are not interpreted, instead they are taken as you type.
So if you hit the "i" key it types "i" instead of changing your mode.

Next you have "Replace" mode.
The only way I know to get into insert mode is by pressing "R" (Capital R). You can also go into temporary replace mode by pressing "r" (lowercase r)
You will know you are in replace mode when you see a small "[REPLACE]" appear at the bottom left of your screen.
When you are in Replace mode, any key you type will replace what is underneath the cursor. So if your cursor is hovering over a P and you press the T key, that P will be replaced by a T.
"Temporary" replace mode (as I refer to it) will only replace one character (the one under the cursor) then return you to command mode.

Last mode you need to learn is "Visual" mode.
You can get to Visual mode a few different ways, but each way preforms a different type of "visual" task.
"Shift-V" puts you into "Visual Line" mode, this allows you to highlight single characters or full words.
"^V" (This means "Leader V" which by default is CTRL V) puts you into "Visual Block" mode, this allows you to edit whole lines at a time, useful to highlight full blocks of code or text.
You will know your in visual mode when you see "[VISUAL]" appear at the bottom left of your screen.
Visual Line and Visual Block are meant for High-Lighting text, I believe the only purpose of this is to "yank" (aka Copy) text

How to move your cursor in VIM
So, part of the purpose of VIM is to keep your fingers on the keyboard and not leave. It also tries hard to focus on keeping your fingers on the home row (of a QWERTY keyboard).
So instead of using the arrow keys, or even WASD. VIM uses HJKL, which can be very confusing at first.
H goes left
L goes right
J goes down
L goes left
That might seem kind of archaic, but if you actually look at the keys it makes quite a bit of since.
Your index finger sits on the "down" arrow all the time, then next to it is the up arrow. So up and down are in the middle.
Your index finger also hits the left key by moving over a little, and your ring finger hits the right key. The left and right keys are on the outside of the up and down keys.
Can you use the arrow keys? Yes, vim does not prevent you from using your arrow keys, but its advised not to use them by purists.

How to enter commands into VIM

Next you need to know when to enter commands, and how to leave different modes.
As I already said, by default you're in "Command Mode", you have to go into other modes by pressing a key, or a combo of keys.

The "Escape" key (top left most key on all standard keyboards usually marked as "Esc") is meant to get you out anything.
If your ever in a strange mode, or if you just want to go back to command mode from Insert mode. You can hit the trusty esc key to get out of almost any mode.

Every key on the keyboard has some kind of a command tied to it, and most of them have 2 or more commands tied to them, so a command could be used by hitting the key, hitting shift then the key, hitting alt then the key("ALT" or <ALT>), hitting ctrl then the key ("^" or "CTRL" or <CTRL>), or hitting "leader" then the key which is "\" by default (<leader>).

Now you need to know about ":" or "colon" commands.
Colon commands can only be entered from command mode. You enter them by typing the colon key (":") then what ever the command is after the colon.

Commands we need to know first is how to quit VIM. The following commands all "quit" vim. Key word is "quit" not "exit".
:q, :q!, ZZ
So, thats ":q" I usually hear people pronounce this as "Colon Q". As mentioned above, this is only entered from command mode.
Please note that ZZ is 2 capitol Z's.

Now you need to know how to "write" your file, again key word is "write" not "save".
:w
But what if you want to Write the file and quit? Do you need to run :q then :w? NO! Vim allows for these 2 to be preformed at the same time, you can do this by typing them together. like this:
:wq so you "write" then "quit". Do not try to ":qw" that would "quit" then it would never write the document because it quit...
What if you need to save the file to a different location, or in a different name?
:w ~/other/folder for writing to a different location, or :w differntName.txt to write it as a different name (like "save as").
You can also combine this as :w ~/location/to/a/place/notTheOrgionalName.sh

Thats all you "need" to know now. You need to know how to edit (insert mode above), and how to write and to quit.

Intermediate Commands

So, you understand how to write and how to quit, and how to do those things together! Thats awesome!
Now, we can begin working on some more advanced stuff! Let me begin by saying that you will never fully learn VIM. You can read reviews on vim from people who have been useing it 5, 10, or 20 years and they will all say "I just learned how to do this!". There is so many things in vim its crazy. So many ways to do what your wanting to do, some more efficient, some more "pure".

Moving around in vim is a little stange, but hoepfully your used to the basic J is down K is up thing.
If you have opened a file larger then 5 lines you probably wanted to get to the bottom of the document instantly and you probably wondered if you could do this in VIM. Yes, you can.
G (Shift g) Takes you to the end of the document, easy enough.
What if you wanted to go back to the top?
gg (g 2 times)That is just hitting the g key 2 times with no modifier keys. Again, pretty easy.

Ok, thats great, how about if you want to go to the end of the line?
$ (Shift 4) Takes you to the end of the line and thats it,
OR A (Shift A) Will take you to the ed of the line AND put you into insert mode.
Ok, what if you want to go to the beggining of the line?
0(Zero) will take you to the start of the line

Cool, you can move from the top of the page to the bottom of the page. You can move to the end of the line and the beggining of the line all with 1 or 2 keystrokes! But what if you want to move only 1 "page" at a time?
^b (Ctrl b) Will move you up 1 full "page" of text. The oposite of this is ^f(Ctrl f) which takes you down of 1 full page of text.
You can also go only "half" of a page with ^u (Ctrl u) for up and ^d (Ctrl d) for down

While were talking about movement, I need to teach you about WORDs and words.
Vim thinks of the below as 1 WORD:
http://masammich.technoanomaly.com
But that same sentence is 7 words

WHAT? How does that work?
VIM has 2 more horizonal movements called "WORD" and "word" that you will see. But before I explain the word, you need to learn about what the difference is.
As far as I understand it,
A "word" is any non-blank character(s) that ends in a white space, a tab, a EOL (End of Line) or any other non-keyword character(basically any symobol). One strange exemption to this is that a blank line is also considered a word.
A "WORD" is a sequence of non-blank characters sepearted by white space. Just like a word, an empty line can also be a WORD.
Again, this gets tricky because "://" is considered a single word. Also "..." would be 1 word but ".a." would be 3 words.
So, a few examples of a single "word" are *(), $$$$$$$$$$$$, and &&<>
A few examples of a single WORD are "WORD", "www.google.com", www.google.com and this-is-1-long-WORD
If this needs more clairification, leave me a comment below, ill go over it with you or edit this to make it better.

Ok, so now that you know what a WORD and a word are, you can mainly read the documentation on VIM and understand movements better now!
So, you can now move horizontally with w and W to move forward to the beggining of each word or WORD, you can also use e and E to move to the end of each word or WORD.
If you want to move backwards in this same manner it will be: b for previous word and B for previous WORD.
Your also able to move backwards useing the ^ (Shift 6) key. This brings you to the previous non-blank character.

That pretty much covers the basics of "movement". Up and down full and half pages, left and right words. Also full line or page movements. It might seem stange, but you only need to memorize the keys for a short period of time, you can always do what I did and go find a pretty graphic online that shows all the vim keys you need, print it out and hang it up near your desk or save it on your desktop. I still reference mine for some commadns i dont use often.

PLUG-INS!

So, VIM comes with a lot of features, and things that can be done by default.
But as you have seen above, some times default isn't good enough, we play with settings to make it better or just personalized.
Now, some people said that this still isn't enough! So they submitted various features to the owner of VIM and were rejected, so rather then give up. They (and others) started making "plug-ins" to add functionality on top of default VIM. This action caught on and is now very popular in the VIM community (very few people DONT use plug-ins in fact) and there are now plug-ins to do almost anything you can imagine (and more frequently being created).

How do I install these?

There are several ways to do this. You can do it manually, or you can use a plug-in manager (recommended) to do it all for you.

Manual although there is nothing wrong with doing it this way, if you plan on having more then a small number of plug-ins, it might be a good idea to use a plug-in manager. But I will still list this way for research, and knowledge.

So first you need to create a directory, then a few directories underneath that, I'm not sure if this is 100% needed, but the way I will list here, is the way I was taught, and I always see others doing it this same way (even with a manager).
So, first create a folder called ".vim" at your home directory.
mkdir ~/.vim
Now go into that directory and we need to create several more directory, I'll explain what each one is for so you can not create them or just remove them if you don't want them.
mkdir -p ~/.vim/{autoload,colors,plugged,swaps,spell}
The above will create 5 sub-directory's underneath ~/.vim. Each of these directory has a specific purpose.
autoload - If you want a certain file to load as soon as vim loads (at the same time) this is where you want to put the .vim file. This is mainly used to put what ever plug-in manager you use (you put the manager's .vim file here) so you may or may not want it. I'm leaving it in here because I'm going to assume you want to just learn how it works, not actually doing all this your self.
colors - This is used for all of your color schemes, being 100% honest, I have 179 colorschemes located in this file right now... I don't even know how I got this many... I probably downloaded a zip of them when I was playing with vim and forgot ... (I have several files still here that don't seem to have been touched in ~a year...). I will suggest you wget or git clone your favorite colorscheme into this folder. Once vim sees you have a .vim, it seems to start referencing it from that point on. So for example, I LOVE the Molokai color scheme, so I cloned it into my ~/.vim/colors directory so I can reference it with my .vimrc.
plugged - Put all of your plug-in files here
swaps - You need to set in your .vimrc to use this for swap files. It's more convenient then having random .~filename floating everywhere you forgot to save or quit properly and left a shadow file behind...
spell - If you chose to turn on spell check in vim, this is where it will save your spell file. Which is where it saves all the words you add or tell it to ignore in a little file.

That's about it, you either wget or git clone the .vim file's into the appropriate place. Colorscheme into the "colors" directory, plug-ins into the "plugged" directory, files to load on launch into the "autoload" directory and the others you setup to have files put into automatically.

Manager This is highly recommended, it makes your life far easier due to it takes the process of updating, and placing files where they need to go out of the picture. Please see the individual links on how to install the separate plug-in, the process changes over time, and I don't want to be behind a revision or 2 for how to install.

Vim-Plug
I personally use this manager. I use because it works in both vim and NeoVim.
It installs and updates in parallel.
It allows you to load a specific branch or tag of a plug-in, so you can play with experimental versions of a plug-in

Pathogen
One of the strengths of Pathogen, is that it creates separate folders for every plug-in, and inside of that folder, is everything the plug-in needs to work.
The benefit of this is that 1 plug-in won't effect the others you have.
Another benefit of this is that if you want to remove a plug-in, you simply remove that directory that is named after the plug-in.

Vundle
The benefit of Vundle is that it is actually built on top of pathogen, so you get all the benefits of pathogen, but you also get a management interface within vim. This is the most popular vim plug-in manager that I know of, most plug-ins you find on GitHub will have vundle instructions on how to use it.
