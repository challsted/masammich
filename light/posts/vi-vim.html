What is VI, Vim, Emacs (and all the others)?

They are command line text editors. they do everything a normal notepad or gedit will do, but with out a GUI installed. It's all done right there in the terminal/shell! A lot of people think its scary/difficult and I'm not going to lie, it's not EASY. But if you force your self to edit all config files and small documents (nothing big and fancy) with one of them, you will learn them. I use VI at work and Vim at home (Vim is just Vi with extra stuff by default). As i mentioned above, Emacs is also a major text editor that a lot of people use and there are also many others that claim to have various features that make it easier to use. In my opinion, if you don't know how to use any of them, then they are all equally easy to learn because no matter what, your learning from scratch. But that's my opinion.
>Why do I chose VI/Vim instead of another one?
Well, as I said, I use it at work, so I have learned to use it pretty well. Hitting :q or :wq is second nature to me now. (quit, and write and then quit respectively).

Whats so good about Vi/Vim and how do i use it?

Whats so good about Vi/Vim?
It's really handy because it loads instantly, even giant files load in less than a second from my experience. Although the commands seem cryptic at first, they actually make sense and are not that difficult to remember usually. Quite a few full-blown IDE's have vi mode that is either built-in or is configurable. Which allows you to use Vi commands in order to edit text. It's very hard to give justice to a text editor that was invented before the mouse was commonly used (and doesn't use it either).

As you can see, this is quite full featured, and this is all just default functionality. You also have a .vimrc file that is kept in your profile and another copy is up in /etc. These can be used to extend the functionality of Vi and Vim.
Another thing you can do is if you have vi or vim installed on your machine (99% of Linux machines i have logged into have Vi already installed, so it's a good idea to learn just for that reason!) is to run "vimtutor" in a terminal. This is built into Vi/Vim and taught me a lot more about Vi/Vim after i thought i knew a lot. None of it is advanced, and everything they teach you in there has a line for you to test what you just learned on.
