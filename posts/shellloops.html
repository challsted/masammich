The main meat and cheese of most programs are called "Loops"

So, as mentioned in my previous article which is meant as a very basic introduction, this article will be about "loops".
They are called this because they "loop" or iterate though data or instructions until they are no longer true. To be able to understand what I mean by "until they are no longer true", you must understand what is known as "Boolean Logic".
Boolean logic (very simplified) is a "true or false" "1 or 0" "left or right", basically, its always one way or the other. There are very in depth guides on how this works and more details if you want to look it up, but to help you understand the basics. So, boolean logic is very simple, everything is either true or false, and it usually evaluates to 1 or 0. So an example would be:
1 < 0 = 0 (False)
0 > 1 = 0 (False)
0 < 1 = 1 (True)
1 > 0 = 1 (True)
So as an example if loop, if [ i < h ]; then this will only do something "IF "i" is less then "h". So if we set "i" to 1 and "h" to 0. Then it would preform what ever we setup. This is only meant to be very basic, so I think this is all you need for now.

"if" loops

So, we covered an example if loop above, but this is just the tip of what we can do with if loops. As we already know, loops will only do something if they are true. So we can code like this.
if [ $KNOWLEDGE == $POWER ]; then
  print "Science Rules!n"
fi
So, as you might be able to tell from above, I am doing a simple comparison.
Its saying "if the value of knowledge is equal to the value of power then print "Science Rules" with a newline character at the end (enter).
You should also notice the semantics of this, it opens with "if" it then has a bracket [, it then gives the first value that we want to check, which is $KNOWLEDGE, it then asks "is it equal to" with ==, the second value $POWER, end bracket ].
Now, it gets slightly program looking from here on, it has a simi colon! So it says "if this previous is true, then do this!" So it would only print "Science Rules" IF knowledge is equal to power. Other wise it will not do anything.
Then to end it, we reverse the word that opened it, so "if" backwards is "fi". This is a common thing in Linux, but not a standard, so donâ€™t expect it to always be this way.
So what if we want to check if the 2 values are not equal to each other? We can do this several ways, for example, * We can say "not" in the if statement with a "!" like this
if [ ! $KNOWLEDGE == $POWER ]; then
  print "Learn More Stuff!"
fi
Or we can do an "else" statement inside of the "if loop"
if [ $KNOWLEDGE == $POWER ]; then
  print "Science, Learning, Books, thats your calling!"
else
  print "Learn More Stuff!"
fi
The first example above does is sometimes called "not the following" or "the opposite of the following", basically its just checks if the next thing is no, then it will then cause it to be yes. That may be confusing, so lets say it a different way. A == B, B == C, and C != D, so if I ask, is A = C? The answer would be true. Then if i ask, is A not equal to D? The answer would be true. A is NOT equal to D.
The 2nd example above is using an "optional" statement, in this case its an else. Else only works if the "if" is false. So in this case, it will only print "Learn More Stuff" if the 2 values are not equal.
You can also put multiple if statements into a single if
